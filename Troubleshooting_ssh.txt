### SSH ###

If you get any troubles with ssh, connections and so on. This will help, hopefully... :)


###  Error: "Too many authentication failures"
 Cause: SSH is trying too many keys before the correct one, causing the server to block further attempts.  

 Fix:
1. Clear all currently loaded SSH keys:  
   ssh-add -D

2. Manually add only the correct key:  
   ssh-add ~/.ssh/nc-ssh

3. Verify that only one key is loaded:  
   ssh-add -l  
   Output should list only one key.

4. Test SSH manually:  
   ssh -o IdentitiesOnly=yes -J ubuntu@bastion ubuntu@webserver
   -  If this works, update Ansible (see Fix 5 below).  
   -  If it fails, check logs (see Fix 3 below).

---

###  Error: "Connection closed by UNKNOWN port 65535"
 Cause: The web server (194.xxx.xx.xx) is forcefully closing the SSH connection.  

 Fix:
1. SSH into the bastion (ip):
   ssh ubuntu@bastion_ip

2. From the bastion, test SSH to the web server:
   ssh -vvv ubuntu@webserver_ip

3. If SSH fails:
   - The firewall or security rules on webserver_ip might be blocking SSH from bastion_ip.
   - Run this on webserver_ip:
     sudo ufw allow from 198.xx.x.xx to any port 22
     sudo systemctl restart sshd
   - Retry SSH.

---


###  Error: "MaxStartups reached" or "MaxSessions exceeded"
 Cause: The web server is rejecting new SSH connections because too many sessions are active.  

 Fix:
1. SSH into the webserver from the bastion:
   ssh ubuntu@bastion_ip
   ssh ubuntu@webserver_ip

2. Edit the SSH config file:
   sudo nano /etc/ssh/sshd_config

3. Add or modify these lines:
   MaxSessions 10
   MaxStartups 10:30:60

4. Save & exit (CTRL+X, then Y, then ENTER).

5. Restart SSH:
   sudo systemctl restart sshd

6. Retry Ansible.(playbook)

---

###  Make Sure Ansible Uses the Correct SSH Configuration
 Ensure Ansible is using the correct key and ProxyJump or ProxyCommand settings.
 Fix:
Edit inventory.ini:
[bastion]
bs ansible_host=bastion ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/nc-ssh

[webserver]
web ansible_host=webserver_ip ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/nc-ssh ansible_ssh_common_args='-o ProxyJump=ubuntu@bastion -o IdentitiesOnly=yes'

 Test SSH manually again:
ssh -o IdentitiesOnly=yes -J ubuntu@bastion ubuntu@webserver

 Test Ansible connectivity:
ansible web -m ping -i inventory.ini -vvvv

 Run the playbook:
ansible-playbook playbooks/setup_webserver.yml -i inventory.ini -vvvv

---

###  Summary of Fixes
 Fix 1: Clear SSH keys (ssh-add -D) & use only one key.  
 Fix 2: Check firewall rules on webserver_ip.  
 Fix 3: View SSH logs (journalctl -u ssh -n 50).  
 Fix 4: Increase SSH MaxSessions (/etc/ssh/sshd_config).  
 Fix 5: Update Ansible inventory for ProxyJump & keys.
